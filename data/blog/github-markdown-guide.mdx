# Introdução

A análise Markdown e Mdx é suportada via `unified`, e outros pacotes de remark e rehype. `next-mdx-remote` nos permite analisar arquivos `.mdx` e `.md` de uma maneira mais flexível sem mexer no webpack.

A marcação com sabor do GitHub é usada. `mdx-prism` fornece recursos de realce de sintaxe para blocos de código. Aqui está uma demonstração de como tudo parece.

A seguinte planilha de descontos foi adaptada de:https://guides.github.com/features/mastering-markdown/

# O que é Markdown?

Markdown é uma maneira de estilizar texto na web. Você controla a exibição do documento; formatar palavras como negrito ou itálico, adicionar imagens e criar listas são apenas algumas das coisas que podemos fazer com o Markdown. Principalmente, Markdown é apenas um texto regular com alguns caracteres não alfabéticos inseridos, como `#` ou `*`.

# Guia

Aqui está uma visão geral da sintaxe do Markdown que você pode usar em qualquer lugar no GitHub.com ou em seus próprios arquivos de texto.

## Headers

```
# Isto é uma tag H1

## Isto é uma tag H2

#### Isto é uma tag H4
```


## Listas

### Não ordenada

```
- Item 1

  - Item 2b
```

### Ordenada

```
1. Item 1
1. Item 2
1. Item 3
   1. Item 3a
   1. Item 3b
```

1. Item 1
1. Item 2

## Tabelas

Você pode criar tabelas reunindo uma lista de palavras e dividindo-as com hífens `-` (para a primeira linha) e, em seguida, separando cada coluna com uma barra vertical `|`:

```
| Primeiro                | Segundo                 |
| --------------------------- | ---------------------------- |
| Conteúdo da célula 1       | Conteúdo da célula 2          |
| Conteúdo da primeira coluna | Conteúdo da segunda coluna |
```

## Tachado

Qualquer palavra com dois tils (como `~~this~~`) aparecerá ~~riscada~~.
